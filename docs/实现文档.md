## 1. 系统概述

ITAM（IT Asset Management）是一套面向企业内部的 IT 资产全生命周期管理平台，支持：

| 功能模块 | 说明 |
|---|---|
| **服务器管理** | 服务器基础信息、硬件规格、网络信息、部署应用的增删改查 |
| **网络设备管理** | 交换机、路由器、防火墙、负载均衡等网络设备管理 |
| **批量导入** | 单文件/多文件批量导入资产数据（SeverCollect.txt 格式） |
| **导入/导出** | Excel（按公司分 Sheet）和 CSV 格式的数据导入与导出 |
| **仪表盘** | 资产总览、按机房/操作系统/归属统计、最近入库记录 |
| **访问控制** | 可选密码认证（Cookie 会话，8 小时有效期） |
| **审计日志** | 所有写操作自动记录，支持变更前后对比（diff） |
| **版本管理** | 语义化版本号，自动递增，每次备份均携带版本标识 |

**生产模式**：Express 同时提供 REST API 和前端静态文件（Single Entry），统一走 `http://localhost:3001`。

### 2.2 后端分层架构

```
HTTP Request
    │
    ▼
Middleware Pipeline
  ├── requestIdMiddleware   (生成请求追踪 ID)
  ├── helmet               (安全响应头)
  ├── cors                 (跨域控制)
  ├── rateLimit            (限流: 15min/300次)
  ├── versionMiddleware    (API 版本识别)
  ├── accessControl        (认证守卫)
  └── auditMiddleware      (审计日志拦截)
    │
    ▼
Router (routes/v1/)
    │
    ▼
Controller
    │
    ▼
Service (业务逻辑 + Zod 校验)
    │
    ▼
Repository (数据访问层, 事务)
    │
    ▼
Prisma ORM → SQLite
```


## 5. 后端实现

### 5.1 应用入口（src/index.ts）

应用启动时完成以下初始化：

1. **环境校验**（fail-fast）：检查 `DATABASE_URL` 是否配置，生产环境密码长度 ≥ 8 位
2. **中间件注册顺序**：
   ```
   helmet → cors → compression → morgan → rateLimit → body-parser → cookieParser
   → requestId → versionMiddleware → auditMiddleware
   ```
3. **路由挂载**：
   - `/api/v1/*` → v1 路由（当前主路由，需认证）
   - `/api/servers/*`、`/api/networks/*`、`/api/applications/*`、`/api/stats/*` → 旧版路由（向后兼容）
   - `/api/health`、`/api/version`、`/api/auth/login`、`/api/auth/logout` → 无需认证
4. **静态文件服务**：生产环境从 `dist/public` 提供 SPA，所有非 `/api` 路径返回 `index.html`
5. **HTTPS 支持**：检测 `backend/ssl/` 目录下的证书文件，有则启用 HTTPS，否则降级 HTTP
6. **优雅关闭**：监听 `SIGTERM` / `SIGINT`，等待请求处理完成后断开 Prisma 连接并退出

### 5.2 中间件详情

#### 认证中间件（auth.ts）

- 通过环境变量 `ACCESS_PASSWORD` 控制是否启用密码认证
- 认证机制：Cookie `auth_session=authenticated`，`httpOnly`，有效期 8 小时
- 密码比对使用 `crypto.timingSafeEqual` 防时序攻击
- 未配置密码时自动放行（内网免认证部署）

#### 审计日志中间件（audit.ts）

- 所有写操作（POST/PUT/DELETE）和导出操作自动记录
- 日志结构包含：`timestamp`、`requestId`、`action`、`method`、`path`、`ip`、`statusCode`、`duration`、变更 `diff`
- `diff` 字段自动计算变更前后差异（by `setAuditContext`）
- 日志写入 `logs/audit.log`，开发环境通过 `/api/audit-logs` 可查询最近 100 条
- GET 读操作默认不记录（可通过 `AUDIT_LOG_READS=true` 开启）

#### API 版本控制中间件（version.ts）

- 版本提取优先级：URL 路径（`/api/v1/`） > `api-version` Header > 默认（`v1`）
- 当前仅支持 `v1`，不支持的版本返回 400

#### 全局错误处理（errorHandler.ts）

| 错误类型 | HTTP 状态码 | 处理方式 |
|---|---|---|
| AppError（业务错误） | 自定义 statusCode | 返回 message，生产环境不带 stack |
| Prisma P2002（唯一约束冲突） | 409 | 数据冲突提示 |
| Prisma P2025（记录不存在） | 404 | 资源不存在 |
| ZodError（校验失败） | 400 | 参数验证失败，开发环境返回详细错误 |
| 未知错误 | 500 | 生产环境脱敏，只返回"服务器内部错误" |

### 5.3 Repository Pattern

所有数据访问通过 Repository 层统一封装，Service 层不直接调用 Prisma。

**ServerRepository** 核心方法：

| 方法 | 说明 |
|---|---|
| `findById(id)` | 按 ID 查询（含关联数据） |
| `findByName(name)` | 按名称查询（唯一性校验用） |
| `findWithPagination(params)` | 分页查询（支持搜索/筛选/排序） |
| `findCompanies()` | 获取去重公司列表（按拼音首字母排序） |
| `findDatacenters(company?)` | 获取去重机房列表（支持按公司筛选） |
| `findOwners(company?)` | 获取去重归属列表 |
| `findAllForExport()` | 获取全量数据（导出用） |
| `createWithRelations(data)` | 事务创建服务器+网络信息+应用信息 |
| `updateWithRelations(id, data)` | 事务更新服务器（含拼音自动计算） |
| `batchDelete(ids)` | 批量删除 |

**特殊设计——拼音首字母排序**：
中文排序无法依赖 SQLite 原生字典序，因此对 `company` 和 `datacenter` 字段，存储时同步计算拼音首字母（`companyPinyin`、`datacenterPinyin`），并在分页查询时优先按拼音字段排序，同音时再按中文名。

### 5.4 服务层（Service）

服务层负责业务规则校验，包括：

- **Zod Schema 校验**：通过 `validateOrThrow` 统一校验输入
- **唯一性校验**：创建/更新服务器时检查 `name` 是否已存在（409 冲突）
- **关联数据处理**：创建服务器时提取 `networkInfos` 和 `applications`，使用事务原子写入
- **导出逻辑**：`export.service.ts` 使用 ExcelJS 按公司分 Sheet 生成 Excel；也支持 CSV 格式

### 5.5 仪表盘统计（stats/dashboard）

使用 `Promise.all` 并行查询所有统计数据：

- 服务器总数 / 运行中 / 离线 / 维护中
- 应用总数 / 运行中
- 网络设备总数 / 运行中 / 离线
- 按机房分组统计
- 按操作系统分组统计（Top 6）
- 按资产归属分组统计（Top 5）
- 最近入库服务器（Top 5）

## 8. 安全机制

### 8.1 安全响应头（Helmet）

```
Content-Security-Policy: 同源限制，禁止内联脚本，禁止 iframe 嵌入
X-Frame-Options: DENY
HSTS: max-age=31536000; includeSubDomains; preload
X-Content-Type-Options: nosniff
```

### 8.2 CORS 配置

- 生产环境：仅允许 `https://{HOST}:{PORT}` 或环境变量 `CORS_ORIGINS`（逗号分隔）
- 开发环境：允许 `http://localhost:5173`（Vite Dev Server）
- 允许携带 Cookie（`credentials: true`）

### 8.3 请求限流

```
窗口：15 分钟
最大请求数：300 次/IP
超限返回：429 Too Many Requests
```

### 8.4 请求体大小限制

```
JSON / FormData：最大 10MB
```

### 8.5 输入校验（Zod）

所有写入操作通过 Zod Schema 校验，非法数据在 Service 层抛出 400 错误。

### 8.6 数据脱敏

- 生产环境错误响应不包含堆栈信息
- 审计日志仅保留关键字段（id、name、company、model、sn、status、datacenter、cabinet）
- 查询参数中的敏感字段（password、token、key 等）在日志中自动脱敏

### 9.3 increment-version.sh

版本号自动递增脚本，每次备份时调用，确保每份备份对应唯一版本号。

**工作流程**：

```
1. 从 backend/src/index.ts 提取当前版本（APP_VERSION）
2. 验证版本号格式（vX.X.X）
3. 递增 patch 版本号（v4.5.7 → v4.5.8）
4. 更新 backend/src/index.ts 中的版本号
5. 更新 backup.sh 中的版本记录
6. 重新构建后端（npm run build）
7. 重启 PM2 服务（pm2 restart itam-backend）
8. 调用 /api/version 验证新版本生效
9. 如存在 .env 中的 APP_VERSION，同步更新
```

集成版本递增 + 备份的一键脚本，用于自动化备份场景：


### 9.5 update.sh

系统更新脚本，包含 5 个强制步骤：

```
0. 记录更新前版本状态

2. 停止 PM2 服务
3. 构建并部署（前端→后端→复制静态文件）
4. 重启 PM2 服务
5. 四项功能验证（API健康/首页/版本接口/服务器API）
```
