## 📋 功能概述

ITAM 系统现已支持 **同时导入多个资产文件**，并实现了 **资产名称不区分大小写匹配** 功能。

### 主要特性
1. **批量文件导入**：可同时上传多个 SeverCollect.txt 格式文件
2. **不区分大小写匹配**：`Server-01` 和 `server-01` 视为同一资产
3. **智能合并**：重复资产自动更新空字段，避免数据丢失
4. **详细报告**：每个文件的导入结果一目了然
5. **统一处理**：服务器和网络设备均支持不区分大小写匹配

---

## 🔧 技术实现

### 1. 不区分大小写匹配
```typescript
// 服务器导入：使用小写键匹配
const namesLower = parsedServers.map(s => s.name.toLowerCase())
const existingMap = new Map(existingServers.map(s => [s.name.toLowerCase(), s]))

// 网络设备导入：手动小写匹配
const nameLower = name.trim().toLowerCase()
const existing = existingDevices.find(d => d.name.toLowerCase() === nameLower) || null
```

### 2. 批量导入优化
- **查询优化**：使用 `name: { in: namesLower }` 一次性查询所有已存在资产
- **批量创建**：使用 `createMany` 批量插入新资产，减少数据库事务
- **拼音预计算**：批量计算公司/机房拼音，避免重复计算

### 3. 文件支持
- **文件格式**：SeverCollect.txt 标准格式
- **文件数量**：支持同时上传任意数量文件
- **文件大小**：建议单个文件不超过 10MB

## 📊 匹配规则

### 服务器匹配规则
| 场景 | 处理方式 | 示例 |
|------|----------|------|
| 主机名完全匹配（大小写不同） | 视为同一服务器，更新空字段 | `Server-01` ↔ `server-01` |
| 主机名不存在 | 创建新服务器 | 新增记录 |
| 主机名存在但字段为空 | 补充缺失字段 | 补充型号、内存等信息 |
| 主机名存在且字段已有值 | 保留原值，不覆盖 | 不修改已有信息 |

### 网络设备匹配规则
| 场景 | 处理方式 | 示例 |
|------|----------|------|
| 设备名完全匹配（大小写不同） | 视为同一设备，更新空字段 | `Switch-01` ↔ `switch-01` |
| 设备名不存在 | 创建新设备 | 新增记录 |
| 设备名存在但字段为空 | 补充缺失字段 | 补充管理IP、固件版本等 |

## 🚨 注意事项

### 数据安全
1. **备份优先**：导入前请使用 `./backup.sh --verify` 备份系统
2. **回滚机制**：导入失败时提供详细错误信息，可手动回滚
3. **权限控制**：导入接口需要相应权限

### 文件格式
1. **编码要求**：文件必须使用 UTF-8 编码
2. **分隔符**：服务器间使用 `===` 分隔符
3. **字段格式**：字段名后使用 `:` 或 `：` 分隔

### 性能限制
1. **内存限制**：单次导入建议不超过 10,000 条记录
2. **超时设置**：API 默认超时 60 秒，大文件请分片
3. **并发限制**：避免同时发起多次批量导入
