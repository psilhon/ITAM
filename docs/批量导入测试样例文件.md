## 如何使用这些样例文件

### 1. 创建测试文件
1. 在桌面上创建一个新文件夹，命名为 `test-import`
2. 复制上面的内容到不同的文本文件中
3. 保存为：
   - `机房A.txt`
   - `机房B.txt`
   - `机房C.txt`
   - `网络设备.txt`

### 2. 测试批量导入
1. 打开 ITAM 系统：http://localhost:3001
2. 进入"服务器管理"页面
3. 点击"导入资产"按钮
4. 切换到"批量导入（多选文件）"模式
5. 选择上面创建的 4 个文件
6. 观察导入过程和结果

### 3. 预期结果
- **文件总数**: 4
- **成功文件**: 4
- **总服务器数**: 4
- **新建服务器**: 3 (前三个文件)
- **网络设备**: 1 (第四个文件，如果网络设备导入功能正常)

### 4. 测试不区分大小写匹配
创建一个重复但大小写不同的文件：

**重复测试.txt**：
```
=== 服务器 ===
主机名: dc1-svr-001  <!-- 注意：这里是小写 -->
公司名称: 公司A-更新
品牌: Dell-新版
型号: PowerEdge R750
备注: 更新后的服务器信息
```

导入这个文件后，应该：
- 主机名 `dc1-svr-001` 会被识别为与 `DC1-SVR-001` 相同
- 系统会更新现有记录（更新字段而不是新建）
- 结果中显示"更新: 1"而不是"新建: 1"

## 文件格式规范

### 基本规则
- 每个服务器/设备以 `=== 服务器 ===` 或 `=== 网络设备 ===` 开始
- 每行格式：`字段名: 值`
- 字段名后必须跟冒号和空格
- 值可以为空，但不能省略冒号
- 支持多行备注（备注字段可以有换行）

### 必需字段
```
主机名: [必需]
公司名称: [必需]
品牌: [可选]
型号: [可选]
资产状态: [必需] (running/offline/maintenance)
```

### 可选字段

## 故障排除

### 常见错误及解决方法

| 错误信息 | 可能原因 | 解决方法 |
|---------|---------|---------|
| "仅支持 .txt 格式文件" | 文件扩展名不是 .txt | 确保文件保存为 .txt 格式 |
| "文件大小不能超过 5MB" | 文件过大 | 拆分大文件为多个小文件 |
| "导入失败" | 文件格式错误 | 检查文件格式是否符合规范 |
| "主机名不能为空" | 缺少必需字段 | 确保每个服务器都有主机名 |
| "公司名称不能为空" | 缺少必需字段 | 确保每个服务器都有公司名称 |

### 性能优化建议
1. **小批量导入**：每次导入 10-20 个文件
2. **文件大小**：每个文件控制在 1MB 以内
3. **字段简化**：只导入必需字段，减少数据量
4. **分批处理**：按部门、机房或时间分批导入
