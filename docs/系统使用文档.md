# ITAM 系统使用文档

## 1. 访问系统

打开浏览器访问：**http://localhost:3001**

系统默认打开仪表盘页面。左侧为导航栏，底部显示当前版本号。

## 2. 仪表盘

仪表盘展示资产概览，包括：
- 服务器总数及状态分布
- 网络设备总数及类型分布
- 最近操作记录

点击卡片可快速跳转对应列表页面。

## 3. 服务器管理

### 3.1 查看列表

左侧菜单「服务器」进入列表页，支持：
- 分页浏览
- 搜索（名称、IP、机房）
- 排序

### 3.2 添加服务器

点击右上角「添加服务器」按钮，填写表单：
- 名称（必填）
- IP 地址（必填）
- 状态：运行中 / 维护中 / 已停机 / 报废
- 类型：物理机 / 虚拟机 / 云服务器
- 机房、制造商、型号等信息

### 3.3 编辑服务器

点击列表操作列「编辑」按钮，修改服务器信息。

### 3.4 删除服务器

点击列表操作列「删除」按钮。

**注意**：单条删除后 60 秒内可在 Toast 通知右侧点击「撤销」按钮恢复。批量删除不可撤销。

### 3.5 批量导入

点击「导入」按钮，上传 Excel 文件（.xlsx）。

Excel 模板列：
| name | ip | status | type | manufacturer | model | idc |
|------|-----|--------|------|--------------|-------|-----|

### 3.6 批量导出

点击「导出」按钮，下载 Excel 文件。

## 4. 网络设备管理

操作方式与服务器管理类似，支持：
- 添加 / 编辑 / 删除网络设备
- 设备类型：交换机 / 路由器 / 防火墙 / 负载均衡器 / VPN设备 / 其他
- 批量导入 / 导出

## 5. 搜索

顶部搜索框支持快速搜索：
- 服务器名称、IP、机房
- 网络设备名称、管理IP

回车或点击搜索图标执行搜索。

## 6. 刷新

点击右上角刷新按钮重新加载当前列表数据。

## 7. 页面布局

- **侧边栏**：导航菜单（可折叠）
- **顶部栏**：搜索、刷新、用户信息
- **主内容区**：列表、详情、表单
- **底部**：版本号

## 8. 表格操作

| 操作 | 说明 |
|------|------|
| 点击列头排序 | 按该列升序 / 降序排列 |
| 悬停行高亮 | 便于定位当前行 |
| 分页 | 底部翻页控件 |

## 9. 表单输入

- **必填字段**：红色星号标识
- **日期选择**：点击选择日期
- **下拉选择**：点击展开选项
- **保存**：填写完成后点击保存按钮

## 10. 状态说明

### 服务器状态
| 状态 | 说明 |
|------|------|
| 运行中 | 正常运行的服务器 |
| 维护中 | 正在进行维护 |
| 已停机 | 已关机停止服务 |
| 报废 | 已退役不再使用 |

### 网络设备状态
| 状态 | 说明 |
|------|------|
| 运行中 | 正常运行 |
| 维护中 | 正在进行维护 |
| 已故障 | 出现故障 |
| 已退役 | 已退役不再使用 |

## 11. 注意事项

- Excel 导入时确保数据格式正确，日期使用标准格式
- 删除操作需谨慎，批量删除不可撤销
- 带 * 号的字段为必填项

## 12. 常见问题

**Q: 打开页面空白怎么办？**
检查服务状态：`curl http://localhost:3001/api/version`，确认返回正常 JSON。

**Q: 删除服务器后能恢复吗？**
单条删除后 60 秒内可在 Toast 通知右侧点击「撤销」按钮恢复。批量删除不可撤销。

**Q: 如何重置数据库？**
```bash
rm backend/prisma/dev.db
cd backend && npx prisma db push && cd ..
./start-local.sh start
```
注意：重置后所有资产数据将清空。
