# 前端批量导入使用指南

## 功能概述
ITAM 系统现在支持两种资产导入模式：
1. **单文件导入**：传统模式，每次导入一个 `SeverCollect.txt` 文件
2. **批量导入**：新功能，可同时导入多个 `SeverCollect.txt` 文件

## 使用步骤

### 1. 进入导入界面
1. 访问系统首页：http://localhost:3001
2. 点击左侧导航栏的"服务器管理"
3. 在页面右上角点击"导入资产"按钮

### 2. 选择导入模式
弹出导入窗口后，可以看到两种模式选择按钮：

```
┌─────────────────────┬─────────────────────────────┐
│  单文件导入           │  批量导入（多选文件）       │
└─────────────────────┴─────────────────────────────┘
```

- **默认模式**：单文件导入
- **点击切换**：根据需要点击相应按钮

### 3. 批量导入操作步骤

#### 3.1 切换到批量导入模式
1. 点击"批量导入（多选文件）"按钮
2. 按钮会变成蓝色选中状态
3. 文件选择框下方会显示操作提示

#### 3.2 选择多个文件
1. 点击"选择文件"按钮
2. 在文件选择对话框中：
   - **Windows**: 按住 `Ctrl` 键选择多个文件
   - **Mac**: 按住 `Command` (⌘) 键选择多个文件
3. 可以同时选择多个 `.txt` 格式的文件
4. **文件要求**：
   - 格式：必须是 `.txt` 文件
   - 大小：每个文件不超过 5MB
   - 内容：符合 `SeverCollect.txt` 格式规范

#### 3.3 开始导入
1. 选择文件后，系统会自动开始导入
2. 界面上会显示"正在批量导入..."的提示
3. 导入过程中请不要关闭窗口

### 4. 查看导入结果

#### 4.1 汇总统计
导入完成后会显示汇总统计面板：


如果有文件导入失败或出现错误：
- 文件名会显示为红色
- 显示具体的错误信息
- 错误文件数量不计入成功统计

### 5. 常见问题

#### Q: 如何切换回单文件导入模式？
A: 点击"单文件导入"按钮即可切换回来

#### Q: 可以混合选择不同类型的文件吗？
A: 不可以，系统只接受 `.txt` 文件。非 `.txt` 文件会被拒绝并提示错误

#### Q: 单个文件的大小限制是多少？
A: 每个文件最大 5MB，超过此限制的文件会被拒绝

#### Q: 批量导入时会按什么顺序处理文件？
A: 系统会同时读取所有文件内容，但导入顺序可能与选择顺序不同

#### Q: 导入过程中可以取消吗？
A: 目前不支持取消导入操作。请确保选择正确的文件后再开始导入

### 6. 最佳实践

#### 6.1 文件组织建议
```
服务器资产/
├── 2026-Q1/
│   ├── 机房A_202603.txt
│   ├── 机房B_202603.txt
│   └── 机房C_202603.txt
├── 2026-Q2/
│   ├── 机房A_202604.txt
│   └── 机房B_202604.txt
└── ...
```

#### 6.2 分批导入策略
- 建议每次导入不超过 10 个文件
- 对于大量文件，按部门或机房分批导入
- 每次导入后检查结果，确认无误后再继续

#### 6.3 错误处理
1. **文件格式错误**：检查文件内容是否符合规范
2. **重复资产**：系统会自动识别并更新现有记录
3. **网络中断**：重新选择文件再次导入

### 7. 技术说明

#### 7.1 前端实现
- 使用 React `useState` 管理导入状态
- `multiple` 属性控制文件选择模式
- 并行读取多个文件内容
- 详细的进度和结果展示

#### 7.2 后端接口
- 批量导入接口：`POST /api/servers/batch-import`
- 支持多文件处理
- 提供详细的导入报告

#### 7.3 性能优化
- 前端并行读取文件内容
- 后端批量处理减少数据库操作
- 智能缓存避免重复处理
